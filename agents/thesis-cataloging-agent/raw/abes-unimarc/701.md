## Commentaires

NB : Pour le traitement des mentions de responsabilité comportant plus de trois noms de personne ou de collectivité voir la [fiche dédiée](https://documentation.abes.fr/sudoc/regles/Catalogage/Regles_PlusDe3Auteurs.htm)

Cette zone suit les mêmes règles que la zone [700](https://documentation.abes.fr/sudoc/formats/unmb/zones/700.htm)

L'une des sous-zones **$a** ou **$3** est obligatoire.

Le **$3**[ppn de la notice autorité liée] est exclusif des sous-zones alphabétiques.

Lorsqu'un lien est établi vers une autre notice, la sous-zone **$3** *remplace* ces sous-zones.

Après validation de la notice, l'interface de catalogage du Sudoc affiche automatiquement le contenu des sous-zones alphabétiques de la notice autorité liée par le **$3** (c'est ce qu'on appelle l'"expansion" du lien).

NB : contrairement aux informations définies dans le format UNIMARC standard, le **$4** est **obligatoire** dans le Sudoc. Voir [Enregistrement des points d'accès auteur à partir d'octobre 2019](https://documentation.abes.fr/sudoc/regles/Catalogage/index.htm#ChoixAcces).

Le code **000 "Fonction à préciser"** est implémenté dans le Sudoc par l'ABES dans toutes les zones 7XX (sauf 716) dépourvues de $4, à partir de fin mars 2013. Il n'est pas affiché dans l'interface publique du catalogue Sudoc, ni exporté dans les notices lors des transferts réguliers.

**Il doit être remplacé en catalogage courant par un code pertinent de la liste des codes de fonction.**La note [309](https://documentation.abes.fr/sudoc/formats/unmb/zones/309.htm) invitant les catalogueurs à corriger ces codes 000 doit être supprimée après correction.
