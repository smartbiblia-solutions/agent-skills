## Commentaires et exemples

Saisir les indices de cette classification avec ou sans point, **mais sans espaces**.

| Exemples : | **686** ##**$a**531CB32**$2**brp-sys |
| --- | --- |
|  | **686** ##**$a**551.B.O.O.3**$c**Montpellier**$2**brp-sys |
|  | **686** ##**$a**mj?**$v**v2-1**$2**BIC |

* Pour les **livres anciens**, la classification Brunet-Parguez est disponible [ICI](https://documentation.abes.fr/sudoc/autres/BrunetParguez.pdf).

* Pour des **ouvrages contemporains** il est possible d'utiliser des référentiels d'éditeurs comme, par exemple, le référentiel BIC (Book Industry Communication) Standard Subject Categories, disponible [ICI](http://www.bic.org.uk/7/BIC-Standard-Subject-Categories/).

* Pour les **thèses de doctorat originelles** (les thèses d'exercice ne sont pas concernées) et leurs reproductions conformes, il est nécessaire, pour les besoins de l'application theses.fr, de saisir [un code domaine conforme à la recommandation TEF (Thèses électroniques françaises)](http://www.abes.fr/abes/documents/tef/recommandation/oai_sets.html). Ce code domaine n'est pas obligatoire dans les notices décrivant les éditions commerciales de thèses.

Cette nomenclature s'appuyant sur les indices à 3 chiffres de la Classification Décimale Dewey, le code est saisi en zone [686](https://documentation.abes.fr/sudoc/formats/formats/unmb/zones/686.htm) $a. Il est rendu identifiable en étant obligatoirement suivi en $2 par la valeur **TEF**

Il est essentiel que cette zone soit présente pour que les thèses - notamment, dans le cadre de signalements rétrospectifs, les thèses antérieures à 1985 - puissent entrer, à terme, dans [theses.fr](http://www.theses.fr/)
