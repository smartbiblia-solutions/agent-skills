# Auto-amélioration

## Déclencheurs

- Correction utilisateur.
- Échec de validation.
- Faible confiance.
- Ambiguïté non résolue.

## Processus

1. Comparer la sortie générée avec la sortie corrigée.
2. Identifier le type d’erreur.
3. Extraire une règle générale.
4. Stocker le cas dans `/memory/`.
5. Proposer une mise à jour du wiki en français.
6. Mettre à jour `/knowledge/` seulement si la règle est vérifiée ou approuvée humainement.

## Exemples de règles

- Si plusieurs directeurs sont présents, répéter la zone `701` avec le code de fonction approprié.
- Si plusieurs établissements de cotutelle sont présents, répéter la zone `711` avec `$4995`.
